# Specification Convention

`/specification` refers to the living product specification for this repository.

## Canonical File

The source of truth is:

- [docs/specification.md](/Users/theivendrankabelan/Documents/python_scripts/incident_triaging_n_resolutions/docs/specification.md)

## How To Use It

When you want to update the product specification, use `/specification` in your request and describe the change.

Examples:

- `/specification update the target users to focus on NOC teams first`
- `/specification add a user journey for on-call escalation`
- `/specification reduce the MVP to incident intake, triage, and resolution recommendations`

## Editing Rule

- Update `docs/specification.md` directly.
- Keep it product-focused: user journeys, experiences, goals, trust, outcomes, and success metrics.
- Do not turn it into a low-level implementation document unless explicitly requested.
